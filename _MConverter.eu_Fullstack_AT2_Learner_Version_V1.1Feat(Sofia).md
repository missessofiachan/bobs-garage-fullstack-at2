<tr class="odd">
<td><p>Time complexity refers to how the execution time of an algorithm
scales with the size of the input data. It is commonly expressed using
Big O notation (e.g., O(n), O(n²), O(log n)), which describes the upper
bound of growth rate. For sorting algorithms, this measures how the
number of comparisons and swaps increases as the number of elements
grows. For example, Bubble Sort has O(n²) time complexity, meaning its
execution time increases quadratically with input size, while Merge Sort
and Quick Sort have average complexities of O(n log n), making them more
efficient for larger datasets.</p>
<p>Space complexity, on the other hand, measures how much additional
memory an algorithm requires relative to input size. In-place algorithms
like Quick Sort use O(log n) extra space (due to recursion), while
others like Merge Sort need O(n) space to hold temporary arrays.
Stability refers to whether an algorithm preserves the relative order of
records with equal keys. A stable sort ensures that if two items are
equal in value, they appear in the same order after sorting. For
instance, Bubble Sort and Merge Sort are stable, while Quick Sort and
Heap Sort are generally not unless modified.</p></td>

halves, recursively sorts them, and merges them back together. It has a

complexity due to temporary arrays used during merging. Merge Sort is
stable and is often used when stable sorting is required, such as in
